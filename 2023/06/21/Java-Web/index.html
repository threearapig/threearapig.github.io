JavaBean 的保存范围
在 <jsp:useBean> 的指令上存在了一个 scope 属性，表示的是一个 JavaBean 的保存范围。保存的范围一共是四种：

  1. page：保存在一页的范围之中，跳转之后此 JavaBean 无效
  2. request：一个 JavaBean 对象可以保存在一次服务器跳转范围中
  3. session：在一个用户的操作范围中保存，重新打开浏览器的时候才会声明新的 JavaBean
  4. application：在整个服务器上保存，服务器关闭时才会消失

第七章

7.1

Servlet 与 JSP 的区别

  1. 擅长方面
    Servlet 的实现方式是在 Java 代码中嵌入 HTML 代码，编写和修改 HTML 非常不方便，所以适合做流程控制和业务逻辑的处理
    JSP 实现的方式是在 HTML 中嵌入 Java 代码，比较适合页面的显示

  2. 编译过程
    JSP 有 JSP Container 进行自动编译，每一个 JSP 页面都会被 Web 容器编译成一个 Java 类，供 Web 容器调用，并且生成 HTML 页面回馈给用户，无论 JSP 文件被创建还是被修改，都不需要对其编译即可执行
    Servlet 编译需要遵循 Java 的标准，Servlet 在编写完成或修改后没有每重新编译，就不能运行在 Web 容器中

  3. 速度不同
    由于 JSP 页面由 JSP Container 对其进行管理，在每次执行不同内容的动态 JSP 页面时，JSP Container 都需要对其进行自动编译，所以它的效率低于 Servlet 的执行效率。
    而 Servlet 在编译完成之后，则不需要再次编译，可以直接获取及输出动态内容。在 JSP 页面中的内容没有变化的情况下，JSP 页面的编译完成之后，JSP Container 不会再次对 JSP 进行编译

7.7

什么是监听器？

监听器就是一个实现特定接口的普通 Java 程序，这个程序专门用于监听另一个 Java 对象的方法调用或属性改变，当被监听对象发生上述事件后，监听器是某个方法将立即被执行
监听器监听的对象，分别为 request、session、servletContext 三大域对象，被监听对象的状态的变化都会触发监听器。而监听器监听的内容包括域对象的创建与销毁以及监听域对象属性的变化

如何使用？

配置监听器的步骤分为三步：

  1. 写一个监听器类去实现监听器接口
  2. 覆盖监听器的方法
  3. 在 web.xml 中进行配置，完成注册

11.5

关于批处理

当要执行多条 SQL 语句时，就需要建立多个数据库连接，这样将会把时间浪费在数据库连接上。因此 JDBC 提供了批处理的方法用于解决这个问题

批处理需要使用 addBatch() 方法将需要执行的 SQL 同时收集起来，最后调用 executeBatch() 方法同时执行这些 SQL 语句。

executeBatch() 方法返回的是一个 int[] 数组，执行多条 SQL 语句就会返回多少个 int 值，这些 int 值就组成了这个 int[] 数组。但是用 addBatch() 方法添加 select 查询语句，程序会直接出现错误。

JDBC 调用存储过程

在 JDBC API 中提供了调用存储过程的方法，通过 CallableStatement 对象进行操作。CallableStatement 继承于 Statement 接口，主要用于执行数据库中定义的存储过程

11.6

事务具备的四个特性

  1. 原子行
  2. 一致性
  3. 隔离性
  4. 持续性

JDBC 的事务支持

使用事务之前需要先调用 Connection 的 setAutoCommit() 方法关闭自动提交，开启事务。进行事务处理，需要按照如下步骤进行操作：

  1. 取消 Connection 中设置的自动提交方式：connection.setAutoCommit(false)
  2. 如果批处理操作成功，则执行提交事务：connection.commit()
  3. 如果批处理操作失败，则肯定会引发异常，应该在异常处理中让事务进行回滚：connection.rollback()
  4. 如果需要，可以设置 Savepoint：Savepoint sp = connection.setSavepoint()

Connection 提供了设置中间点的方法：setSavepoint()。中间点可以实现更细粒度的控制回滚操作
